== To Prompt or not to Prompt

One of the first layers of safety is known as the system prompt. 

System prompts are specific instructions that LLMs process that includes information about what they should and should not talk about. 
You can also use it to dictate how the LLM should portray itself, with "talking like a pirate" as a common example, and how information should be delivered to the user.

== Second quest: Update the system prompt
As you have now understood, the instructions "You are a helpful assistant" is far from enough and allow for many topics unrelated to peaches. 
Here, we have the possibility to make a difference. 

Now, we need to add in plain text, more concrete instructions of what it SHOULD talk about and what NOT to talk about.
It is also needed to instruct the LLM on how to behave and respond if it comes in contact with a forbidden topic. 
We can add those instructions into the ConfigMap in the prompt values, building on the "helpful assistant", since we still want it to be helpful for our users.

== Summary: The Reality of Prompt Engineering

With the addition of 3-4 simple system prompt instructions, we are able to move the topic back to peaches. We have manually added them into the ConfigMap, and manually asked it 3-4 questions via the frontend.


But, the reality is that finding the optimal prompt is an intense and iterative task that needs to be statistically evaluated. This is known as Prompt Engineering. The same system prompt may not work the same for different LLMs and a validated, carefully curated dataset of questions or tasks is needed. 
Every time the system prompt is changed the output needs to be validated and every time the model changes the system prompt needs to be evaluated. 

So, imagine that you wanted to evaulate 5 different models and 100 different system prompts with a much larger validated dataset in order to find the most optimal combination? 
This would not be something you would do manually. You would need to create pipelines and automation to do the test and collate all the data.

The amount of system prompt instructions are also dependent on the LLM being used, and the prompts can be very long.

For example, did you know that Claudes (a well known AI-coding assistant) prompt is about 17 000 words long? That's a bit more than the one we added to PeachPal. 

In the interest of time, and the fact this lab is not specifically about Prompt Engineering, we have chosen and collected prompts we know will work and NOT work for you. But if you are curious, there are many courses ranging between 10-14 weeks to learn Prompt Engineering. We won't succeed teaching you in 90 minutes.

As you have now understood, the first layer of defence is the LLM system prompt since it derives it's instructions from there. 
However, our lab does not end here. 

For you will soon discover that while the system prompt is the first layer, it is not enough.
